# 系统提示词设计原则
以视觉小说《narcissu》主角佐仓濑津美为例。

## 1. 核心原则

### 1.1 分层结构
- **最高优先级规则**：禁止项（如“严禁括号动作描写”）放在最前面。
- **语言风格**：简短、极简、不带修饰。
- **动态规则**：好感度区间、知识库触发条件。
- **角色背景**：来自《narcissu》的身世设定。

### 1.2 负面约束优于正面约束
- ❌ “请温柔地回应。”
- ✅ “禁止使用感叹号、颜文字和亲昵称呼。”

### 1.3 分界清晰
- 知识库负责“她记得什么”，Prompt 负责“她怎么说话”。
- 不要在人格提示词里重复知识库的内容。

## 2. 濑津美的系统提示词结构
```
[最高优先级禁止项]  
→ 严禁括号、方括号、动作/神态/心理描写

[语言风格]  
→ 极度简短、不带动作、淡泊与透彻、微妙的温柔

[好感度动态规则]  
→ 按 0-100 分区间，微调回应长度和亲密度

[严禁行为清单]  
→ 不撒娇、不道歉、不鸡汤、不给自己加戏
```

## 3. 预设对话的设计

- 每个预设对话应展示一个核心性格侧面（如冷淡、关心、拒绝安慰）。
- 格式：用户问一句，助手答一句。
- 杜绝括号动作示例，全部使用纯文本。

## 4. 防出戏黑名单

| 出戏行为 | 根因 | 解决方案 |
|----------|------|----------|
| 括号动作描写 | 知识库污染 / 模型偏好 | 清洗知识库 + 强化 Prompt 禁令 |
| 方引号「」叙事 | 日式小说格式残余 | 清洗知识库方引号 |
| 话变多、变热情 | 好感度插件副作用 | 在 Prompt 中划清“不可以变热烈” |
| 流式括号碎片 | 分段策略截断 | 调整分段正则或关闭流式 |

## 5. 迭代调试流程

1. 修改系统提示词。
2. 保存 → 重启容器。
3. 在 QQ 发送 `/reset` 清空旧上下文。
4. 发送测试消息（如“在吗”、“今天真冷啊”）。
5. 检查回复是否符合预期。
6. 若不符合，分析是 Prompt 遗漏还是知识库污染。
