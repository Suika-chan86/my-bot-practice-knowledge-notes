# 知识库 + 长期记忆 + Prompt 的协同

## 1. 三层信息注入模型

| 层次 | 来源 | 注入时机 | 特点 |
|------|------|----------|------|
| 系统提示词 (Prompt) | 你编写的角色设定 | 每轮对话都完整携带 | 稳定但容量有限 |
| 知识库 (RAG) | 《narcissu》剧本等外部文档 | 用户提问时实时检索 | 精准但依赖检索质量 |
| 长期记忆 | APLR 自动总结的过往对话 | 每次请求时检索注入 | 动态更新，赋予角色连续性 |

## 2. 三者如何不冲突？
- Prompt 负责“她怎么说话”，不需要包含剧情细节。
- 知识库负责“她经历过什么”，所有剧情事实放入文档交给检索。
- 长期记忆负责“她和用户之间发生过什么”，避免每次聊天像初见。

## 3. 注入优先级与冲突处理
- 系统提示词中的禁令高于一切。
- 知识库检索到的内容如果包含动作描写，会被清洗（预处理时剥离）。
- 长期记忆如果被总结出不符合人设的句子（如“她说我今天很开心”），可以通过调整总结提示词的风格指令来改善。

## 4. 协同配置示例
- Prompt 中写：“你可以参考记忆和知识库来理解对话背景，但表达方式必须始终遵循你的语言风格。”
- 知识库设置中，关闭 `Agentic 知识库检索`（测试阶段），强制每次检索，防止角色自行决定忽略。
- 长期记忆插件中将“记忆注入方式”设为 `user_prompt`，并采用 `prepend` 前置注入，让记忆作为上下文出现在对话最前面而不污染系统级设定。

## 5. 常见协同故障
| 症状 | 可能原因 | 排查方向 |
|------|----------|----------|
| 角色忘记刚说过的话 | 长期记忆总结间隔太长 | 调低 `num_pairs` 或手动触发总结 |
| 角色说出知识库原文但不简练 | 知识库检索返回过多 | 降低 `最终返回结果数` |
| 角色突然照搬剧本动作 | 知识库未清洗或 Prompt 未加禁令 | 重新清洗并强化禁令 |
| 氛围割裂 | 长期记忆注入位置不对 | 改为 `prepend` 并控制记忆数量 |
