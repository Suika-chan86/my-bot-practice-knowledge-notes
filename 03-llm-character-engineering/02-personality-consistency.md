# 人格一致性与防动作描写策略

## 1. 一致性问题的根源
LLM 本质上是概率模型，在长对话中会逐渐“遗忘”最初的设定，尤其是角色话少时更容易被用户带偏。此外，知识库中的原文若带有括号动作或心理描写，模型会不自觉地模仿。

## 2. 保障一致性的三层防线

| 层级 | 机制 | 作用 |
|------|------|------|
| 最高优先级禁令 | 系统提示词开头明确禁止动作描写、括号、颜文字 | 从源头约束生成行为 |
| 知识库清洗 | 上传前用 sed 剥离所有括号内容 | 避免模型从知识库中习得出戏元素 |
| 定期重置 + 测试 | 每次修改配置后 `/reset` 并发送测试句 | 快速发现并修复人格漂移 |

## 3. 防动作描写的具体措施
- **系统提示词首部**：  
[最高优先级规则]  
你的回复中严禁出现任何描写动作、神态、心理活动的文字，
包括但不限于全角括号（）、方引号「」、星号**等包裹的内容。
你只有纯粹的语言输出，没有旁白。
- **知识库预处理**：使用 sed 命令删除全角括号及其内容。
- **内容过滤正则**（兜底开关）：在 AstrBot 的“内容过滤正则表达式”中填入 `[（）]`，可强制屏蔽漏网的括号字符。

## 4. 角色漂移的常见表现

| 漂移类型 | 表现 | 原因 | 修复 |
|----------|------|------|------|
| 笑容/动作描写 | （轻轻笑了） | 知识库残留 | 清洗知识库 |
| 突然热情 | 用感叹号或颜文字 | 好感度插件配置不当 | Prompt 中加入“无论好感度多高，不允许多话” |
| 叙事化方括号 | 「这样吗」 | 日语引号格式经模型模仿 | 清洗知识库中方括号 |
| 上下文被路人污染 | 对不相关的人说出亲昵语句 | 未开启“隔离会话” | 在设置中打开 `unique_session` |

## 5. 人格维护检查表
- [ ] 系统提示词开头是否有最高优先级的禁止项？
- [ ] 知识库文件是否已剥离全角/半角括号？
- [ ] 是否开启“内容过滤正则”作为兜底？
- [ ] 是否开启“隔离会话”？
- [ ] 每次修改配置是否 `/reset` 并测试了至少 5 条对话？
