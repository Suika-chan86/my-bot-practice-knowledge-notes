# 知识库 RAG 原理

## 1. RAG 是什么？

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将外部知识文档与 LLM 结合的技术。

```
用户提问 → Embedding 向量化 → 向量数据库检索 → 重排序 → 注入 Prompt → LLM 生成回复
```

## 3. 核心步骤

### 3.1 文档分块（Chunking）
- 将长文档切成固定大小的文本块。
- 参数：`分块大小`（如 512 字符）、`分块重叠`（如 50 字符）。

### 3.2 向量化（Embedding）
- 将文本块转换为固定维度的向量（如 1024 维）。
- 模型：`BAAI/bge-m3`（硅基流动免费提供）。

### 3.3 相似度检索
- 用户提问也被向量化，在向量数据库中搜索最相似的文本块。
- 参数：`融合检索结果数`（如 20）、`最终返回结果数`（如 3）。

### 3.4 重排序（Rerank）
- 对初步检索结果进行更精细的相关性排序。
- 模型：`BAAI/bge-reranker-v2-m3`（硅基流动免费提供）。

## 4. 实践要点

### 4.1 防动作描写污染
- 剧本中含大量括号动作描写（如“（轻轻摇头）”）。
- 需用 `sed` 清洗中文括号：`sed -i 's/（[^）]*）//g' 文件.txt`。
- 同理清洗方括号：`sed -i 's/「[^」]*」//g' 文件.txt`。

### 4.2 调整检索参数
- 角色话极少时，降低 `最终返回结果数`（如设为 3），避免信息过载。
- 测试阶段关闭 `Agentic 知识库检索`，强制每次检索。

### 4.3 Embedding 模型配置
- 硅基流动 `BAAI/bge-m3`：免费、中文效果好、维度 1024。
- 配置路径：服务提供商 → 添加 OpenAI Embedding → 填入 API Key、Base URL、模型名。

### 4.4 Rerank 模型配置
- 服务提供商 → 添加重排序 → 选择 vLLM Rerank。
- API Base URL 只填 `https://api.siliconflow.cn/v1`，不要加 `/rerank`。
- 模型名：`BAAI/bge-reranker-v2-m3`。
